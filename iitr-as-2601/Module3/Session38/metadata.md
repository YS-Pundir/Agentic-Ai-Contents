Course Name: Certification in Agentic Systems and Design

Target Audience : Students from any backgorund may not be necessarily form tech background


session duration: 2hr 15 mins

title: RAG Foundations

objective: Understand how to give AI its own 'Library' — the big picture before building retrieval in code.

topics be covered: Why RAG?; The five-step RAG flow; Retriever, generator, and grounding; Text to embeddings (session demo); Bridge to vector DB and full RAG (upcoming sessions)


detailed subtopics to be covered:
* Explain briefly why LLMs alone are not enough for organization-specific or up-to-date questions (knowledge cutoff, hallucination risk);
* Define Retrieval-Augmented Generation (RAG) as giving the model a searchable external “library” instead of relying only on training memory;
* Walk through the five-step RAG flow once at a conceptual level: ingest → prepare → retrieve → augment prompt → generate;
* Explain retriever vs generator and grounding in plain language (answer from supplied context when the library has the fact);
* Compare one question answered without external context vs with context pasted from a sample document (manual demo — no full RAG app in this session);
* Demonstrate how input text converts to embeddings using Ollama (extend the embeddings preview from the previous session);
* Explain that embeddings will be stored in a vector database and, in upcoming sessions, retrieved and fed to the LLM to complete the RAG loop.

Note for instructors — teach lightly; defer to later sessions:
* RAG vs fine-tuning (deep comparison), nine-stage pipeline detail, chunking/top-k tuning, failure-mode drills, full coded RAG pipeline — upcoming topics: Embeddings & Vector Search; Building a RAG App.
