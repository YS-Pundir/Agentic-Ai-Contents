## Introduction: Why Repetition Needs Automation
Imagine sending the same message to 100 people.

You wouldn’t type it 100 times.  
You’d want a way to say: *“Do this again and again.”*

In programming, this repetition is handled using **loops**.

A loop tells the computer:
> “Repeat this task until I say stop.”

---

## Why Loops Matter
Loops are essential because they let programs:
1. **Save time** by avoiding repeated code  
2. **Scale easily** from 2 items to 2 million  
3. **Handle collections of data** naturally  

Most real programs use loops constantly.

---

## From Known to New: Repeating Actions → Loops
### The Manual Way
You might write the same line of code many times:
- Print item 1  
- Print item 2  
- Print item 3  

This is slow and error-prone.

### The Loop Way
You tell Python:
- “For each item, do this”  
- “While this condition is true, keep going”  

One block of code replaces many repeated lines.

---

## Core Loop Concepts
### 1. For Loops
A **for loop** repeats an action for each item in a sequence.

- Commonly used with lists  
- Stops automatically when items end  

*Think: “For every item, do this.”*

---

### 2. While Loops
A **while loop** repeats as long as a condition stays true.

- Useful when you don’t know the number of repetitions  
- Requires careful conditions to avoid infinite loops  

*Think: “Keep going while this is true.”*

---

### 3. Iterating Over Lists
A **list** holds multiple values.

Loops let you:
- Visit each item one by one  
- Apply the same logic to all items  

This is how programs process collections of data.

---

### 4. Loop Control: Break and Continue
Sometimes you need control inside a loop.

- **break** → exit the loop immediately  
- **continue** → skip the current step and move on  

They help handle special cases cleanly.

---

## Putting It All Together
A typical loop flow
1. Start a loop  
2. Process each item or condition  
3. Skip or stop when needed  
4. Finish automatically or by break  

This is the backbone of automation.
