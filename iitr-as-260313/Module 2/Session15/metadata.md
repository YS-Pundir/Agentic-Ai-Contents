lecture ID: 152511

Course Name: Certification in Agentic Systems and Design

Target Audience : Students from any backgorund may not be necessarily form tech background


session duration: 2hr 15 mins

title: Using SQL Databases with AI Applications

objective: Learn how AI applications access and query database systems

topics be covered: querying databases;database access patterns for agents


detailed subtopics to be covered:
- Recall Core SQL Operations — Quick recap of CRUD operations and basic SELECT queries from Session 14; bridge into more complex querying;
- Apply Sorting, Limiting, and Pagination — Use ORDER BY (ASC/DESC), LIMIT, and OFFSET to control result sets; connect this to practical use cases like displaying top results or paginated lists;
- Understand Database Relationships — Explain the concept of related tables, why data is normalized across multiple tables, and how relationships eliminate redundancy;
- Learn Key Relationship Terminologies — Define one-to-one, one-to-many, many-to-many relationships; primary keys, foreign keys, and referential integrity with real-world examples;
- Design and Create Related Tables in Supabase — Build two related tables (e.g., users and orders) with a foreign key constraint; visualize the schema in Supabase's table editor;
- Query Relational Data with JOINs — Introduce INNER JOIN using a two-table example (users and orders); write a hands-on query that combines both tables to retrieve meaningful results (e.g., which user placed which order); understand what a JOIN does conceptually before writing it — limited to two-table joins only;
- Perform CRUD on Related Data — Insert, update, and delete records across related tables while maintaining referential integrity; understand cascading behavior with simple examples;
