# Subjective Assignment: Build a Reusable LangChain Explanation Chain

## Practical Task (Medium)

A learning-support assistant needs to explain technical topics to different audiences in different tones. Build a single Python program that uses a reusable **PromptTemplate**, sends it to an Ollama model through **ChatOllama**, and returns clean text using **StrOutputParser**.

### Requirements

Write a Python program named `learning_explainer.py` that:

1. Imports `PromptTemplate` from `langchain_core.prompts`.
2. Imports `StrOutputParser` from `langchain_core.output_parsers`.
3. Imports `ChatOllama` from `langchain_ollama`.
4. Creates a `ChatOllama` model with:
   - `model="qwen2.5:0.5b"`
   - `base_url="http://localhost:11434"`
   - `temperature=0.3`
5. Takes four inputs from the user:
   - topic
   - audience
   - tone
   - word limit
6. Creates a `PromptTemplate` using placeholders:
   - `{topic}`
   - `{audience}`
   - `{tone}`
   - `{limit}`
7. The prompt should ask the model to:
   - explain the topic to the selected audience
   - use the selected tone
   - include one real-life analogy
   - stay within the selected word limit
8. Creates a `StrOutputParser`.
9. Creates an LCEL chain in this exact order:

```python
prompt | llm | output_parser
```

10. Runs the chain using `chain.invoke({...})`.
11. Prints the final answer returned by the chain.

### Input Format

The program should take four inputs, one per line:

```text
topic
audience
tone
word_limit
```

### Example Input


## Answer Explanation

### Ideal Solution Walkthrough

The program should first import the LangChain building blocks. `PromptTemplate` creates a reusable prompt with variables, `ChatOllama` connects the program to the local Ollama model, and `StrOutputParser` converts the model response into a plain string.

The four user inputs become runtime values for the prompt placeholders. The LCEL pipe connects the prompt, LLM, and parser in the correct order, so the chain runs as one flow.

### Complete Code

```python
# Import PromptTemplate to create a reusable prompt with placeholders
from langchain_core.prompts import PromptTemplate

# Import StrOutputParser to convert model output into a plain string
from langchain_core.output_parsers import StrOutputParser

# Import ChatOllama to connect LangChain with the local Ollama model
from langchain_ollama import ChatOllama

# Take the topic from the user
topic = input("Enter topic: ")

# Take the audience from the user
audience = input("Enter audience: ")

# Take the tone from the user
tone = input("Enter tone: ")

# Take the word limit from the user
limit = input("Enter word limit: ")

# Create the LangChain wrapper for the local Ollama model
llm = ChatOllama(
    model="qwen2.5:0.5b",  # Use the local model name available in ollama list
    base_url="http://localhost:11434",  # Connect to the local Ollama server
    temperature=0.3,  # Keep the answer focused and stable
)

# Create a reusable prompt template with four placeholders
prompt = PromptTemplate.from_template(
    """Explain {topic} to {audience} with these requirements:
- Use {tone} tone
- Give one real-life analogy
- Keep the answer within {limit} words"""
)

# Create the output parser that returns only plain text
output_parser = StrOutputParser()

# Connect prompt, model, and parser using the LCEL pipe operator
chain = prompt | llm | output_parser

# Run the chain by passing values for all template placeholders
result = chain.invoke({
    "topic": topic,  # Fill the topic placeholder
    "audience": audience,  # Fill the audience placeholder
    "tone": tone,  # Fill the tone placeholder
    "limit": limit,  # Fill the limit placeholder
})

# Print the final clean string returned by StrOutputParser
print(result)
```

### Alternative Approaches

- The input values can be hard-coded for a quick test, but user input is better for a reusable program.
- The same chain can explain other topics by changing only the input values.
- A different local Ollama model can be used if the `model` name is also changed in the code and the model is available in `ollama list`.
