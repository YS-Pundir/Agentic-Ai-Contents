| Regularization - intuition behind Ridge and Lasso regression | covered | Regularization was explained as reducing model power to reduce overfitting. Ridge/L2 was covered as shrinking large coefficients toward zero, while Lasso/L1 was covered as a stricter method that can push unimportant feature coefficients to zero. |
| Model Comparison - impact of regularization on model behavior | covered | Linear regression, Ridge, and Lasso were compared using train/test scores and coefficient behavior. The session discussed how regularization reduced overfitting, improved test performance, and made the model more generalizable. |
| Evaluation Metrics - MAE RMSE and R-squared interpretation | covered | R-squared was explained through `.score()` and model accuracy/variance explanation. MAE was explained as mean absolute error using actual minus predicted residuals, and RMSE was explained as root mean squared error that penalizes larger mistakes more strongly. |
| Error Analysis - identifying patterns in prediction errors | covered | Error analysis was covered mainly through residuals/errors, actual minus predicted values, and how error magnitude affects MAE/RMSE. There was limited focus on deeper residual-pattern diagnosis such as plotting or systematic error pattern detection. |
| Practical Perspective - choosing metrics and evaluating model performance | covered | The session discussed using R-squared for model score/accuracy, MAE/RMSE for interpreting prediction error, RMSE when large errors matter more, and test metrics as the most important before deployment. |
| Extra concept taught: train/test split, overfitting, underfitting, and good fit recap | covered | The instructor spent time revising train/test evaluation, unseen test data, high train vs low test performance as overfitting, high train/test error as underfitting, and the idea of a better good-fit model. |
| Extra concept taught: coefficients and residuals | covered | Coefficients were revisited as model weights, and residuals were repeatedly explained as the difference between actual and predicted values. |
| Extra concept taught: alpha as regularization strength and hyperparameter tuning | covered | Alpha was explained as the parameter controlling regularization strength. The session showed that alpha equal to zero means no regularization, higher alpha increases regularization, and choosing the right alpha is a hyperparameter tuning process. |
| Extra concept taught: Lasso as feature selection | covered | Lasso was emphasized as an automatic feature selector because it can reduce unimportant feature coefficients to zero and effectively remove weak features from the model. |
| Extra concept taught: practical sklearn implementation | covered | The session demonstrated or discussed using `LinearRegression`, `Ridge`, `Lasso`, `.fit()`, `.score()`, `.predict()`, and metric functions such as mean absolute error / mean squared error style utilities. |
